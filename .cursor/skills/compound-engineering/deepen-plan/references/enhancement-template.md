# Enhancement Output Templates

## Section Enhancement Format

For each plan section, append research insights in this format:

```markdown
## [Original Section Title]

[Original content preserved — do NOT modify]

### Research Insights

**Best Practices:**

- [Concrete recommendation with source]
- [Concrete recommendation with source]

**Performance Considerations:**

- [Optimization opportunity]

**Edge Cases:**

- [Edge case and how to handle it]

**References:**

- [Documentation URL or source]
```

## Enhancement Summary Format

Add this block at the **top** of the plan file, before the first section:

```markdown
## Enhancement Summary

**Deepened on:** [YYYY-MM-DD]
**Sections enhanced:** [Count]
**Research agents dispatched:** [List of agent types used]

### Key Improvements

1. [Major improvement 1]
2. [Major improvement 2]
```

## Quality Checklist

Before writing the enhanced plan, verify:

- [ ] All original content preserved — no deletions or modifications to existing text
- [ ] Research insights clearly marked under `### Research Insights` subsections
- [ ] Each recommendation is concrete and actionable (no vague "consider doing X")
- [ ] Code examples (if any) are syntactically correct
- [ ] No contradictions between enhanced sections
- [ ] Enhancement Summary accurately reflects all changes
- [ ] References use valid URLs or verifiable source names
