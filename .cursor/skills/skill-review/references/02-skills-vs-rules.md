## 2. Skills vs Rules：如何选择

Cursor 提供两种 Agent 行为定制机制：**Rules（**`**.mdc**`** 文件）** 和 **Skills（**`**SKILL.md**`**）**。选择错误的机制会导致无效或低效的行为控制。

### 对比

| 维度           | Rules (`.mdc`)                      | Skills (`SKILL.md`)               |
| -------------- | ----------------------------------- | --------------------------------- |
| **触发方式**   | `alwaysApply: true` 或 `globs` 匹配 | Agent 根据 `description` 智能判断 |
| **加载时机**   | 总是加载 / 文件匹配时加载           | 仅在相关时按需加载                |
| **适用内容**   | 编码标准、格式要求、命名约定        | 复杂工作流、多步骤流程、决策逻辑  |
| **内容风格**   | 声明式规则（"应该"/"不应该"）       | 操作式指令（步骤1、步骤2...）     |
| **上下文成本** | 始终占用 token（alwaysApply 时）    | 按需加载，不用时零成本            |

### 选择决策

```plaintext
需要定制 Agent 行为？
├── 所有文件都需要遵守？ → Rule (alwaysApply: true)
│   例：版权头、TODO 格式、安全规则
├── 特定文件类型需要遵守？ → Rule (globs)
│   例：C++ 编码规范 → globs: "**/*.{h,cc}"
├── 需要多步骤流程？ → Skill
│   例：代码审查流程、告警添加流程
├── 需要决策逻辑？ → Skill
│   例：根据输入类型选择不同的处理方式
└── 需要交互式确认？ → Skill
    例：执行前需要用户确认参数
```

### 迁移信号

当你发现一个 Rule 变得越来越复杂（包含流程步骤、条件分支、需要用户交互），就是将其迁移为 Skill 的信号。
