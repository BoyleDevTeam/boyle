## 11. 发布前检查清单

### 核心质量

- [ ] `name` 仅使用小写字母、数字、连字符，≤64 字符
- [ ] `description` 具体且包含触发关键词，≤1024 字符
- [ ] `description` 使用第三人称
- [ ] `description` 描述**何时使用**而非**工作流概要**
- [ ] `SKILL.md` 正文不超过 500 行
- [ ] 全文术语一致
- [ ] 示例具体，非抽象

### 结构

- [ ] 辅助文件引用不超过一层深度
- [ ] 适当使用渐进式展开
- [ ] 工作流有清晰的步骤编号
- [ ] 不包含时效性信息
- [ ] 不使用 Windows 路径格式

### 如果包含脚本

- [ ] 脚本自行处理错误，而非推给 Agent
- [ ] 必需的依赖已在指令中说明
- [ ] 错误处理明确且有帮助性信息
- [ ] 明确区分"执行"和"阅读参考"

### Cursor 特有功能

- [ ] 需要进度追踪时集成了 TodoWrite
- [ ] 需要用户输入时使用了 AskQuestion 门控
- [ ] 仅显式触发的 Skill 设置了 `disable-model-invocation: true`
- [ ] 跨技能引用使用了 skill 名称（而非文件路径）

### 测试

- [ ] 在至少 3 个场景中测试过
- [ ] Agent 能正确发现和激活此 Skill
- [ ] Agent 按预期遵循指令
- [ ] 已识别并封堵常见的"合理化借口"
