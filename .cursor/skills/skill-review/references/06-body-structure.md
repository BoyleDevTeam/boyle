## 6. SKILL.md 正文结构

### 推荐的章节组织

```markdown
---
name: skill-name
description: ...
---

# Skill Title

## Overview

核心原则，1-2 句话。

## When to Use / When NOT to Use

触发场景和排除场景（如果决策不明显，用小型流程图）。

## Checklist / Workflow

明确的步骤列表或检查清单。

## Quick Reference

表格或要点列表，便于快速扫描。

## Common Mistakes

常见错误和修复方式。

## Reference Files

指向辅助文件的链接。
```

### 章节规模匹配复杂度

不要为每个章节写同样多的内容。简单的方面用一两句话，复杂的方面详细展开：
"Scale each section to its complexity: a few sentences if straightforward, up to 200-300 words if nuanced."
